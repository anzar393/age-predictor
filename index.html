WiFi Pentesting Basics

WiFi pentesting involves assessing wireless networks for vulnerabilities like weak encryption (e.g., WEP, WPA2) or misconfigurations. Common goals: detect rogue access points, test password strength, or identify open ports.

Key concepts: SSID, BSSID, encryption types (WPA3 is strongest), and tools like packet sniffing.

Recommended Tools (Install on Kali Linux VM)

  • Aircrack-ng: For capturing and cracking WiFi handshakes.
  • Wireshark: Packet analyzer for monitoring traffic.
  • Reaver: For WPS-enabled routers (if vulnerable).
  • Fern WiFi Cracker: GUI-based tool for scanning and attacking.

Note: These are open-source; use ethically and legally.

High-Level Pentesting Steps (On Your Own Network)

  1. Reconnaissance: Scan for nearby networks using tools like `airodump-ng` to list SSIDs and channels.
  2. Monitor Mode: Put your WiFi adapter into monitor mode to capture packets.
  3. Capture Handshake: Deauthenticate a client to force a reconnection and capture the 4-way handshake.
  4. Crack Password: Use dictionary attacks (e.g., with `aircrack-ng`) on the captured data. (Never use on others' networks.)
  5. Report Findings: Document vulnerabilities and suggest fixes like enabling WPA3 or changing default passwords.

Example command (do not run without permission): `sudo airodump-ng wlan0mon`
